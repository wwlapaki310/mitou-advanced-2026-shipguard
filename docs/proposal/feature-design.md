# 機能設計書

## アプリ全体の機能構成

ShipGuardは**2つの装着モード**を持つ。

```
┌─────────────────────────────────────────────────────────────┐
│                        ShipGuard                            │
│                                                             │
│  ┌─────────────────────┐   ┌─────────────────────────────┐  │
│  │   船体固定モード      │   │    おでこ装着モード           │  │
│  │  （スタンドorホルダー）│   │  （ヘッドバンド/ヘルメット）  │  │
│  │                     │   │                             │  │
│  │  ① 外界検知          │   │  ④ 視線連動検知              │  │
│  │    障害物・他船       │   │    首振りIMU                │  │
│  │    浮遊物・岸壁       │   │    + カメラ（目線方向）      │  │
│  │                     │   │    → 見た先にある物を検知    │  │
│  │  ② 転覆危険度        │   │    → 信号旗・形象物・船舶   │  │
│  │    ジャイロ+加速度    │   │    → 音声で即時通知         │  │
│  │    HUD表示          │   │                             │  │
│  └─────────────────────┘   │  ⑤ 安全確認行動モニタリング  │  │
│                             │    見張り義務の実施を検知    │  │
│  ┌─────────────────────┐   │    前方/後方/転舵方向確認    │  │
│  │  ③ ローカルLLM相談   │   │    未実施なら音声ガイド      │  │
│  │  （停泊中・待機時）   │   └─────────────────────────────┘  │
│  └─────────────────────┘                                    │
└─────────────────────────────────────────────────────────────┘
```

## 装着モード詳細

### 船体固定モード
- スマホをスタンドやマウントホルダーで船体に固定
- 常時前方を向いたままカメラが稼働
- ①外界検知 + ②転覆危険度が動作
- 操縦者はスクリーンを「ちらっと見る」程度でOK

### おでこ装着モード（視線連動）
- ヘッドバンド型マウント or ヘルメット取付型にスマホを固定
- カメラが**操縦者の視線方向**を常に向く
- ④視線連動検知 + ⑤安全確認行動モニタリングが動作
- 操縦者は**スクリーンを見ずに音声だけで情報を受け取る**

## 機能④：視線連動 障害物・信号旗 検知（おでこモード）

### コンセプト
おでこ装着することで「カメラ＝操縦者が今見ている方向」になる。
**振り向いた先に何があるかを自動認識し、音声で即時通知する。**

操縦者はスクリーンを見る必要がなく、視線を外さずに情報を得られる。

### 検知対象

#### A. 他船・障害物
| 検知対象 | 音声通知例 |
|---|---|
| 他船（大型） | 「右前方に大型船接近、約200m」 |
| 他船（小型） | 「左前方に小型船」 |
| 浮遊物・ブイ | 「正面に浮遊物」 |
| 泳者・落水者 | 「前方に人！　速度を落として」 |
| 岸壁・防波堤 | 「前方に岸壁接近、注意」 |

#### B. 信号旗（国際信号旗）の検知・読み上げ
他船に掲げられた信号旗を検知し、意味を音声で通知する。
免許取得者でも旗の意味を即座に判断するのは難しく、教育的価値も高い。

| 旗 | 意味 | 音声通知例 |
|---|---|---|
| **B旗**（青赤縦二色） | 危険物取扱中 | 「前方の船、危険物取扱中。接近注意」 |
| **Q旗**（黄色） | 検疫旗（出入港時） | 「前方の船、検疫旗（Q旗）を掲揚中」 |
| **O旗** | 人身事故発生 | 「前方の船、人身事故旗（O旗）。118番通報を確認」 |
| **N旗＋C旗** | 遭難信号 | 「警告！遭難信号を検知。海上保安庁118番へ」 |

#### C. 形象物（Day Shapes）の検知
昼間に船舶が掲げる形象物（球・円錐・ひし形など）を検知し、意味を通知する。

| 形象物 | 意味 | 音声通知例 |
|---|---|---|
| 球形2個（縦） | 操縦不能船 | 「前方の船、操縦不能。回避してください」 |
| 球形3個（縦） | 運転制限船 | 「前方の船、運転制限中。注意」 |
| 円錐形（頂点下向き） | 帆走中の動力船 | 「前方、帆走中の動力船」 |
| 球形+菱形+球形 | 引き船（曳航長さ200m超） | 「前方、長い曳航作業中。広めに回避」 |
| アンカーボール（球1個） | 錨泊中 | 「前方の船、停泊中（錨泊）」 |

### システム構成

```
おでこカメラ映像
  ↓
物体検知（YOLOv8 Fine-tuning）
  ├─ 船舶クラス → 距離・方向推定
  ├─ 信号旗クラス → 旗の種類分類 → 意味データベースで解決
  └─ 形象物クラス → 形状認識 → 意味データベースで解決
  ↓
IMU（頭部方向）と融合
  └─ 「右45°、距離150mの船にB旗」のような空間的な情報へ
  ↓
音声合成（Text-to-Speech / flutter_tts）
  └─ 操縦者が画面を見ずに情報を受け取る
```

### 信号旗・形象物検知の実装アプローチ

```
課題：信号旗・形象物の学習データが少ない

対策：
  1. 合成データ生成（Blender / Stable Diffusionで旗画像を生成）
  2. 海事資料・教本の図版画像をアノテーション
  3. 実港湾・マリーナでの実撮影
  4. 色ベースの前処理フィルタ（旗の色・形状で候補絞り込み）で精度補助
```

## 機能⑤：安全確認行動モニタリング（おでこモード）

### コンセプト
首の動き（Yaw・Pitch変化）を検知して、**見張り義務の実施を自動記録・フィードバック**する。

### 検知する確認行動

| 確認行動 | 頭部動作 | トリガー条件 |
|---|---|---|
| 前方確認 | 正面を向く（基準姿勢への復帰） | 30秒以上未確認 |
| 左確認 | Yaw +60°以上 | 左転舵前3秒以内 |
| 右確認 | Yaw −60°以上 | 右転舵前3秒以内 |
| 後方確認 | Yaw ±150°以上 | 5分以上未確認 |
| 下方確認 | Pitch −20°以上（水面・障害物） | 必要に応じて |

### 音声ガイドのフロー

```
[通常航行中、30秒以上前方未確認]
→ 「前方を確認してください」（音声）

[左転舵を検知、転舵前3秒に左確認がない]
→ 「左方向を確認しましたか？」（音声）

[5分以上後方未確認]
→ 「後方の確認をお願いします」（音声）
```

### 操縦ログとしての記録
- 確認行動の実施/未実施を時系列ログとして保存
- 操縦後に「安全確認スコア」としてフィードバック（教習・練習用途）
- 免許スクールでの実技練習ツールとしての活用に向く

## 音声通知システム設計

### 優先度制御

```
緊急（赤）: 即時割り込み再生
  例：「警告！遭難信号を検知」「落水者！」
  
注意（黄）: 現在の音声終了後に再生
  例：「右前方に船舶接近」「ロール角30度超過」

情報（緑）: キューに追加、5秒以上沈黙時に再生
  例：「前方の船、錨泊中」「後方を確認してください」
```

### TTS実装（Flutter）
```dart
// flutter_tts パッケージ
final tts = FlutterTts();
await tts.setLanguage("ja-JP");
await tts.setSpeechRate(1.2); // 少し速め（緊急感）
await tts.speak("前方に船舶接近");
```
