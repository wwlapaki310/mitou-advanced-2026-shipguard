# 評価方法・テスト計画

## 評価の全体方針

実海域テストは費用・安全・再現性の面でハードルが高い。
**段階的評価（室内 → 模擬環境 → 実海域）** によって、
低コスト・高再現性で精度検証を進める。

### ② IMU転覆危険度モジュール

#### 核心的課題
実船で意図的に傾けてテストするのは危険・高コスト・再現性が低い。

#### 評価環境：遊園地の船系アトラクション活用

| アトラクション | 計測できる動き | 評価対象 |
|---|---|---|
| **バイキング（海賊船）** | 大きなロール角変化（±30〜45°程度） | ロール角検知・警告タイミング |
| **ウォーターライド・激流下り** | 急加速・急減速・落下衝撃 | 横G・縦G の閾値超過検知 |
| **スプラッシュ系（落下あり）** | 急激な縦方向衝撃 | 衝撃イベント検知 |
| **回転系アトラクション** | ヨー方向の急回転 | 急旋回検知 |

**メリット**
- 繰り返し同条件で計測可能（再現性◎）
- 実際の転覆前動作に近い大きな傾きを安全に体験できる
- 入場料のみで低コスト（1日で大量データ収集可能）
- 季節・天候に依存しない

**手順**
1. スマホをアトラクションの手すり・座席に固定（マウントクランプ等）
2. IMUログ（加速度・ジャイロ・推定Roll/Pitch角）を記録
3. 映像（GoPro等）と同期させて「実際の傾き角」と「推定角」を比較
4. 警告が適切なタイミングで発火するか確認

**評価指標**
| 指標 | 目標値 |
|---|---|
| ロール角推定誤差（バイキング静止端点） | ±5°以内 |
| 危険度警告の発火タイミング（遅延） | ≤ 0.5秒 |
| 誤警告率（水平移動時） | ≤ 5% |
| 実船での主観評価（n=5名） | 「違和感ない」≥ 80% |

### ④【新機能】安全確認行動モニタリングモジュール

#### コンセプト
**スマホをおでこ（額）に装着**し、操縦者の頭部回転をIMUで検知することで、
適切な安全確認行動（見張り義務）が実施されているかをリアルタイムで判定する。

```
装着イメージ：
  ヘッドバンド型マウント or ヘルメット取付型マウントに
  スマホを固定 → 首の動きがそのままIMUデータになる

検知対象の安全確認行動：
  ・前方確認  → Pitch角が下向きから水平に戻る動作
  ・後方確認  → Yaw角が±150〜180°回転する動作
  ・転舵前確認 → 転舵方向（左 or 右）へのYaw回転イベント
```

#### 背景・根拠
- 海上衝突予防法 第5条：「すべての船舶は、常時、適切な見張りをしなければならない」
- 事故分析で「見張り不十分」が原因上位に挙がる
- 自動車のドライバーモニタリングシステム（DMS）の船舶版として位置づけ可能

#### 安全確認フロー検知ロジック
```
[通常航行中]
  → 30秒ごとに「前方確認イベント」を期待
  → 未実施なら「確認してください」音声ガイド

[転舵前]
  → 舵角センサー（将来）または加速度パターンで旋回開始を検知
  → 転舵方向への頭部回転が「転舵前3秒以内」にあるかチェック
  → なければ警告

[後方確認]
  → 5分ごとに後方確認イベントを期待
```

#### 評価方法

**Step 1：室内シミュレーション評価**
- おでこにスマホを装着し、椅子に座って首を各方向に意図的に動かす
- Yaw±90°（左右）、Yaw±180°（後方）、Pitch変化（前方）を正しく検知できるか確認
- 評価指標：各方向の検知True Positive率 ≥ 90%

**Step 2：遊園地アトラクション評価**
- 激流下りや船系アトラクションに乗りながら装着
- 波・振動・急加速など「ノイズ」環境下での誤検知（False Positive）率を計測
- 評価指標：ノイズ環境での誤検知 ≤ 10%

**Step 3：実船評価**
- 実際の操縦者（免許保有者 n=5〜10名）に装着してもらい実船で操縦
- 評価者が「確認した・しなかった」を別途記録し、アプリの検知と突合
- 評価指標：確認行動の検知精度（Precision/Recall） ≥ 0.80

**評価指標まとめ**
| 指標 | 目標値 |
|---|---|
| 各方向の確認イベント検知精度（Precision） | ≥ 0.85 |
| 確認イベント再現率（Recall） | ≥ 0.80 |
| ノイズ環境での誤検知率（FP） | ≤ 10% |
| 音声ガイドへの操縦者の反応率 | ≥ 70%（ユーザーテストで主観調査） |
