## 技術スタック確定版

| 層 | 選定技術 | バージョン目安 | 選定理由 |
|---|---|---|---|
| フレームワーク | Flutter | 3.x | iOS/Android統一・カメラ/センサーAPIが充実 |
| 言語 | Dart + C/C++（FFI） | — | LLM推論はllama.cppをFFI経由で呼び出し |
| 物体検知モデル | YOLOv8n → TFLite/CoreML | Ultralytics 8.x | FPS最優先・エコシステム充実 |
| モバイル推論（検知） | TFLite Runtime / CoreML | — | Android/iOS両対応 |
| IMU処理 | sensors_plus + Dart実装 | — | 50Hz処理に十分・純Dart実装 |
| 姿勢フィルタ | Complementary Filter | — | 軽量・実績あり（将来Madgwickに移行） |
| LLMモデル | Gemma 3 2B IT (Q4_K_M) | 〜1.5GB | 日本語対応・スマホ実績あり |
| LLM推論エンジン | llama.cpp (JNI/Swift FFI) | latest | クロスプラットフォーム・最軽量 |
| RAGベクトルDB | sqlite-vss | — | モバイル軽量・SQLite互換 |
| CI/CD | GitHub Actions | — | テスト自動化・APKビルド |

## リスク管理

| リスク | 発生確率 | 影響度 | 対策 |
|---|---|---|---|
| FPS不足（15 FPS未達） | 中 | 高 | 入力解像度削減・フレームスキップ・モデル軽量化（YOLOv8n → カスタム） |
| 海上データ不足（精度不足） | 中 | 高 | Roboflow公開データ活用・Data Augmentation（海上光反射シミュレーション） |
| IMUドリフト（姿勢推定精度） | 低 | 中 | 定期キャリブレーション・Madgwick Filter移行 |
| LLMモデルサイズ（配布困難） | 低 | 中 | アプリ内課金方式でオンDL・Qwen 1.5Bに切替 |
| 誤警告率が高い（信頼性低下） | 中 | 高 | 閾値調整UIをユーザーに開放・クラス別信頼度表示 |
| 発熱・バッテリー消費 | 高 | 中 | 推論頻度を可変設定・バックグラウンド省電力モード |
| 法的責任（過信・事故） | 低 | 高 | 「補助ツール」明示・免責事項・UX設計で操縦中LLM無効化 |
