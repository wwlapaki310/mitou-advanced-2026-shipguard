### 空白地帯（ShipGuardの差分）

**既存アプリはすべてAIS/GPS情報（=登録済み船舶）のみを扱う。**
- 浮遊物・転覆船・漁具・波・岸壁などの**非AIS障害物はゼロ検知**
- スマホIMUを使った**姿勢・転覆危険度**を提供するアプリは国内外に存在しない
- 船舶向けの**オフライン動作LLM相談**機能を持つアプリは存在しない
- **3機能を統合したアプリ**は確認できず（→未踏性の核心）

## 3. 学術研究の動向

### 海上物体検知（Maritime Object Detection）

- **Singapore Maritime Dataset (SMD)** や **SeaShips** などのベンチマークで海上CV研究が活発
- YOLOv8/v9 ベースの海上物体検知は精度面で実用水準に達している（2023〜2024）
- ただし研究の多くは**サーバーGPU前提**。スマホエッジ上でのリアルタイム動作に特化した研究は少ない
- VLM（Vision Language Model）を海上に適用する研究もあるが**推論速度が実用的でない**（5〜10 FPS未満が課題）

→ 本提案は「FPS最優先 × スマホオンデバイス × 船舶特化クラス」という組み合わせで差別化

### IMUベース転覆検知

- 船舶向けIMUの学術研究は主に**専用ハードウェア（FOG・高精度MEMS）**前提
- スマホの民生用MEMS IMUをリアルタイム転覆危険度判定に使う実装は確認できず
- 補完フィルタ（Complementary Filter）やMadgwickフィルタを使えば民生MEMSでもロール・ピッチ推定が可能

### オンデバイスLLM

- Gemma 3（2B）、Qwen2.5（1.5B〜）が2025年以降スマホ上での実用推論に対応
- MediaPipe LLM Inference API・llama.cpp（Android/iOS）・Ollama モバイル版 が利用可能
- **海事ドメイン特化のオンデバイスLLMアプリは存在しない**（→RAG+量子化モデルの組み合わせが新規）

## 4. 特許調査（J-PlatPat）

- 「船舶 障害物検知 スマートフォン」：関連特許なし（2025年時点）
- 「船舶 転覆検知 加速度センサ」：専用ブイ・センサー搭載前提の特許複数あり。スマホ単体での実装特許なし
- → 本提案の実装方式での特許リスクは現時点で低い（引き続き精査が必要）

---

## 5. 結論：未踏性の根拠

本提案が新規である理由を3点に集約できる：

1. **カメラ × IMU × LLMの統合**：3機能を一つのスマホアプリで提供する先行例なし
2. **非AIS障害物へのアプローチ**：既存アプリはAIS登録船しか扱えない。浮遊物・転覆危険など現実の事故原因をカバーできていない
3. **0円後付けモデル**：既存のすべての小型船舶に対して追加HWなしで展開可能。270万人の免許保有者に即日リーチできる唯一のアプローチ
